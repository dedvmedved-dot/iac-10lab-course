# ЛР5 — Верификация кластера PostgreSQL + Patroni

**Статус**: 🔴 КРИТИЧЕСКИЙ
**Дата**: 2026-06-25

## Ключевой результат

ЛР5 НЕ СОДЕРЖИТ Terraform-код вообще — ни main.tf, ни variables.tf. Книга только перечисляет их в структуре каталогов (стр. 654). Студент без кода построить 7 ВМ не сможет.

### Найденные проблемы

#### 1. 🔴🔴🔴 CRITICAL: Отсутствует весь Terraform-код

Книга приводит структуру:
```
terraform/
├── main.tf          # НЕТ в книге
├── variables.tf     # НЕТ в книге
├── outputs.tf       # НЕТ в книге
└── terraform.tfvars.example  # НЕТ в книге
```

Ни одного файла не предоставлено. Для 7 ВМ это критический пробел.

#### 2. 🔴 CRITICAL: `ansible_user=student` (сквозной)

Везде в inventory.ini — `student`, а облачный образ создаёт `ubuntu`.

#### 3. 🟡 MEDIUM: 7 ВМ — перегрузка Proxmox

Одновременное создание 7 ВМ через Terraform гарантированно приведёт к таймаутам и падению провайдера (проверено на 4 ВМ в ЛР4).

**Рекомендация**: `-parallelism=2`

#### 4. 🟡 MEDIUM: Пароли в открытом виде в group_vars

`postgres_superuser_password: "postgrespass"` — не учебная практика, ансибл-волт не упомянут.

#### 5. 🟢 LOW: etcd v3.4 устарел

Используется etcd v3.4 (EOL 2022), актуальная v3.5. Для учебных целей допустимо, но стоит отметить.

### Рекомендации

1. Добавить полный main.tf с `clone`, `datastore_id=vm-storage`, `user=ubuntu`
2. Заменить `student` → `ubuntu`
3. Указать `-parallelism=2`
4. Примечание про ansible-vault для продакшена
