## Найденные проблемы

### 1. 🔴 CRITICAL: Terraform-код не приведён

В отличие от ЛР1/ЛР3, в ЛР4 нет готового main.tf. Студент должен написать его сам.
При этом книга не описывает критически важные детали:

- блок `clone { vm_id = 9000 }` — без него ВМ не создаются
- `node_name` должно совпадать с именем узла Proxmox (`pve`, не `proxmox`)
- `datastore_id` зависит от конфигурации стенда (`vm-storage` vs `local-lvm`)

**Исправление**: в репозиторий добавлен рабочий main.tf с `for_each` (4 ВМ).

### 2. 🔴 CRITICAL: VIP 192.168.0.200 конфликтует с Proxmox

```yaml
# group_vars/all.yml в книге:
vip_address: "192.168.0.200"
```

`192.168.0.200` — это IP-адрес самого сервера Proxmox. Назначение VIP на этот адрес приведёт к:
- потере доступа к Proxmox WUI/API
- конфликту ARP
- недоступности всех ВМ через API

**Исправление**: VIP заменён на `192.168.0.210`.

### 3. 🔴 CRITICAL: Несовпадение пользователя `student` vs `ubuntu`

Как и в ЛР1-ЛР3. Облачный образ Ubuntu создаёт `ubuntu`, а не `student`.

| Где | В книге | Должно быть |
|-----|---------|-------------|
| inventory.ini | `ansible_user=student` | `ansible_user=ubuntu` |
| SSH-команды | `ssh student@...` | `ssh ubuntu@...` |

### 4. 🟢 LOW: Интерфейс — книга предупреждает, но не даёт команду проверки

Книга справедливо отмечает (раздел 4.8.3): «Имя интерфейса eth0 может отличаться. В некоторых ВМ сетевой интерфейс называется ens18, ens19, enp0s3». Рекомендация верная, но нет конкретной команды проверки.

**Рекомендация**: добавить `ip -br link show | awk '{print $1}'` в раздел настройки Keepalived.

### 5. 🟡 MEDIUM: Отсутствует `ANSIBLE_HOST_KEY_CHECKING=False`

Книга не упоминает эту переменную. При первом подключении Ansible откажется соединяться без неё.

### 6. 🟡 MEDIUM: `proxmox_node_name` не документирован

Книга не показывает, как узнать имя узла Proxmox. В реальных стендах оно может быть `pve`, `proxmox`, `node1` и т.д.

Команда для проверки:
```bash
pvesh get /nodes --output-format json | jq '.[].node'
```

### 7. 🟢 LOW: PEP 668 для установки Ansible

Не упомянута проблема с `pip install ansible` на современных дистрибутивах.

### 8. 🟢 LOW: Повтор IP-адреса web-01 с предыдущими ЛР

Та же проблема, что между ЛР1 и ЛР3.
