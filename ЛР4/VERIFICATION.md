# ЛР4 — Верификация HA + балансировка (HAProxy + Keepalived + VIP)

**Статус**: 🔴 КРИТИЧЕСКИЙ — найдены ошибки, блокирующие выполнение
**Дата**: 2026-06-25
**Стенд**: Proxmox pve 9.2.3, OpenTofu 1.9.1

## Ключевой результат

ЛР4 содержит ОТСУТСТВИЕ Terraform-кода и КРИТИЧЕСКИЙ конфликт VIP с хостом Proxmox. Без исправления этих проблем лабораторная невыполнима.

### Найденные проблемы

#### 1. 🔴🔴🔴 CRITICAL: VIP конфликтует с Proxmox (БЛОКИРУЕТ)

| Адрес | Назначение |
|-------|-----------|
| 192.168.0.200 | Proxmox VE (управление) |
| 192.168.0.200 | VIP для балансировщиков (в книге) |

Установка VIP на тот же IP, который использует Proxmox, приведёт к коллизии, недоступности Proxmox Web UI и API, либо к невозможности поднять VIP.

**Исправление**: использовать другой адрес, например 192.168.0.210.

#### 2. 🔴 CRITICAL: Отсутствует Terraform-код

Книга говорит «в Terraform/OpenTofu нужно описать четыре ВМ», но НЕ ПРИВОДИТ код. Студент вынужден:
- Либо писать с нуля (высокий порог входа)
- Либо копировать из ЛР1/ЛР3 (копирование багов)
- Либо искать в комплекте (отсутствует)

#### 3. 🔴 CRITICAL: Несовпадение пользователя (сквозной баг)

Везде `ansible_user=student`, но облачный образ создаёт `ubuntu`.

#### 4. 🟡 MEDIUM: Интерфейс `eth0` не универсален

Книга признаёт проблему в сноске, но не даёт решение. В Ubuntu cloud-images интерфейс — `ens18`.

**Решение**: `vip_interface: "{{ ansible_default_ipv4.interface }}"` — Ansible определит автоматически.

#### 5. 🟡 MEDIUM: `local-lvm` в примере таблицы

В таблице параметров указано `local-lvm`, которого нет на ZFS-стендах.

#### 6. 🟡 MEDIUM: 4 параллельные ВМ могут перегрузить Proxmox

Создание 4 ВМ через Terraform параллельно может привести к таймаутам и падению провайдера.

**Рекомендация**: `-parallelism=2` или создание последовательно.

#### 7. 🟢 LOW: bash-синтаксис в книге

Команда `for i in {1..6}` работает в bash, но не в `/bin/sh`. В ОС, где `/bin/sh` — это dash, конструкция `{1..6}` не работает.

**Решение**: `for i in $(seq 1 6)`

## Рекомендации

1. **Заменить VIP**: `192.168.0.200` → `192.168.0.210` (или другой свободный)
2. **Добавить Terraform main.tf** — с блоком `clone`, корректным `datastore_id`, пользователем `ubuntu`
3. **Унифицировать пользователя**
4. **Использовать `ansible_default_ipv4.interface`** для VIP-интерфейса
5. Рекомендовать `-parallelism=2`
