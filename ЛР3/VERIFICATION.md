### Найденные проблемы

#### 1. 🔴 CRITICAL: main.tf без `clone` (повтор ЛР1)

В примере main.tf нет блока клонирования из шаблона. Без него `terraform apply` завершится ошибкой.

**Исправление**: добавлен блок `clone { vm_id = 9000; full = true }`

#### 2. 🔴 CRITICAL: Несовпадение пользователя (повтор ЛР1, ЛР2)

| Где | Пользователь |
|-----|-------------|
| main.tf variables | `student` |
| Исходный образ | `ubuntu` |
| inventory.ini | `student` |

В учебных файлах везде указан `student`, но облачный образ Ubuntu создаёт `ubuntu`.

**Исправление**: везде заменён на `ubuntu`.

#### 3. 🔴 NEW: Конфликт VM ID с существующими ВМ

VM ID 201 был занят ВМ `k8s-master1`. В учебном стенде такие конфликты возникают, если студент не очищает предыдущие лабораторные. Книга не предупреждает об этом.

**Рекомендация**: добавить в `terraform.tfvars.example` комментарий: «Убедитесь, что VM ID не заняты: qm list»

#### 4. 🟡 MEDIUM: `datastore_id = "local-lvm"` не существует

В стенде хранилища: `local`, `local-zfs`, `vm-storage`. Имя `local-lvm` — стандартное для установок Proxmox на LVM, но на ZFS-стендах оно будет `local-zfs`.

**Исправление**: заменено на `vm-storage`.

#### 5. 🟡 MEDIUM: Повтор IP-адреса web-01 между ЛР1 и ЛР3

ЛР1: web-01 = 192.168.0.101 (ID 105)
ЛР3: web-01 = 192.168.0.101 (ID 201)

Один и тот же IP на разных ВМ. Нужно либо уничтожать старую ВМ, либо использовать разные IP.

#### 6. 🟡 MEDIUM: Неупомянута установка провайдера

В разделе «Запуск Terraform» нет информации, что `terraform init` может не найти провайдер из-за гео-блокировок реестра HashiCorp. Необходим filesystem mirror или альтернативное решение.

#### 7. 🟢 LOW: `outputs.tf` содержит захардкоженный `student`

Строки вида `"ssh student@192.168.0.101"` — должны использовать переменную `var.vm_user`.

## Ansible-часть (замечания те же, что в ЛР2)

Playbook не разворачивался полностью из-за совпадения шаблона с ЛР2, но ожидаемые проблемы:
- `ansible_user=student` — та же проблема несовпадения пользователя
- `ANSIBLE_HOST_KEY_CHECKING` не упомянут
- PEP 668 для установки Ansible на современных дистрибутивах

## Рекомендации

1. Добавить блок `clone` во все main.tf примеры
2. Унифицировать пользователя (`student` через cloud-init user-data или `ubuntu` через inventory)
3. Указать необходимость проверки VM ID перед запуском
4. Добавить примечание о возможной смене `datastore_id`
5. Указать разные IP для разных лабораторных или требование destroy перед новой главой
