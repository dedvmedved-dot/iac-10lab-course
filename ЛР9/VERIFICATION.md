# ЛР9 — Верификация Proxmox + Terraform/OpenTofu

**Статус**: 🔴 КРИТИЧЕСКИЙ — базовая лаборатория имеет те же баги что и ЛР1
**Дата**: 2026-06-25

## Ключевой результат

ЛР9 — это «правильный» способ создания шаблона и массовых ВМ. Но код в книге содержит ВСЕ сквозные баги.

### Найденные проблемы

#### 1. 🔴🔴🔴 CRITICAL: `local-lvm` жёстко задано (15 упоминаний!)

Книга НИ РАЗУ не упоминает `vm-storage`. Все пути: `local-lvm:vm-9000-disk-0`, `--storage local-lvm`.

**Книга**:
```hcl
variable "vm_datastore_id" {
  default = "local-lvm"
}
```

На стендах Proxmox 9.x с ZFS хранилище `vm-storage`, а не `local-lvm`.

#### 2. 🔴 CRITICAL: `student` везде (9 упоминаний)

`ansible_user=student` жёстко задан. На Ubuntu cloud-images пользователь — `ubuntu`.

#### 3. 🟡 MEDIUM: Нет `clone` блока для TOFU

Книга показывает РУЧНОЕ создание шаблона через `qm create + qm importdisk + qm set`, затем отдельно `tofu apply` для массовых ВМ из шаблона. Это двухфазный процесс.

Но в самом Terraform-коде для массовых ВМ тоже нет `clone` — он создаёт их через ссылку на шаблон, но код не показан.

**Исправление**: добавить полный main.tf с блоком:
```hcl
clone {
  vm_id = 9000
  full  = true
}
```

#### 4. 🟡 LOW: Команды в книге ожидают `student`, а не `ubuntu`

Примеры подключения: `ssh student@192.168.0.X` — не будут работать.

### Рекомендации

Переписать ЛР9 с учётом:
1. `vm-storage` вместо `local-lvm`
2. `ubuntu` вместо `student`
3. Полный main.tf с `clone` блоком
4. `cloud-init` с пользователем `ubuntu`
